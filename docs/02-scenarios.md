# 02 - 二十個完整使用情境

## 標記說明

- `[審]` 關鍵動作需手機審核（ntfy Web Push，寄信 / 金錢 / 發布 / 刪除）
- `[排]` 可排程自動觸發
- `[本]` 需本機執行（federation 派發）
- 觸發來源：LibreChat Web UI（電腦）/ 排程 / OneAI PWA（手機）

每個情境格式：觸發 → 流程 → 支柱 → 產出 → 驗收。

---

## A. 提案 / 業務開發

### S01 客戶提案自動草擬
- 觸發：Web UI 輸入客戶與需求。
- 流程：RAG 撈 vault 過往提案/SOP → persona 去 AI 味撰寫 → 產出投影片大綱與文件。
- 支柱：Brain + Manager。
- 產出：`.pptx` + `.docx`。
- 驗收：內容引用過往案例、語氣符合 persona、可直接小幅修改即用。

### S02 報價單生成 `[審]`
- 觸發：Web UI 給專案範圍。
- 流程：依範圍 × vault 歷史報價估算 → 產出報價表 → 金額送手機審核。
- 支柱：Brain + Guardrail。
- 產出：報價表（data / doc）。
- 驗收：金額正確、審核通過後才定版。

### S03 客戶來信回覆草稿 `[審]`
- 觸發：Email 進來（排程輪詢）或 Web UI 指定。
- 流程：讀信脈絡 → persona 草稿 → 手機審核 → 寄出。
- 支柱：Brain + Guardrail。
- 產出：已寄出的回信。
- 驗收：未經審核不得寄出；審核後送達。

### S04 客戶會議前簡報包
- 觸發：行事曆偵測到會議（排程）或手動。
- 流程：研究客戶背景（web + vault）→ 整理重點 → 產簡報。
- 支柱：Brain + Manager。
- 產出：`.pptx` + 一頁 brief。
- 驗收：會議前 N 小時備妥並推播提醒。

---

## B. 內容創作

### S05 Threads / 社群貼文草擬 `[審]`
- 觸發：Web UI 或排程主題。
- 流程：persona 語氣產 3 版草稿 → 手機選稿/審核 → 發布。
- 支柱：Brain + Guardrail。
- 產出：貼文文字（發布前停）。
- 驗收：發布需審核；語氣去 AI 味。

### S06 部落格長文
- 觸發：Web UI 給題目。
- 流程：deep research → 大綱 → persona 草稿 → 校對。
- 產出：Markdown / `.docx` / 網頁。
- 驗收：附引用、結構完整、語氣一致。

### S07 Landing Page 生成 `[審-發布]`
- 觸發：Web UI 給產品描述。
- 流程：文案 + 版面 → 本機生成可部署網頁 → 預覽 → 發布。
- 支柱：Brain + Hands。
- 產出：靜態網頁。
- 驗收：本機可預覽；公開發布需審核。

### S08 既有簡報改版美化
- 觸發：上傳舊 `.pptx`。
- 流程：解析 → 重設計版面/文案 → 輸出新版。
- 產出：`.pptx`。
- 驗收：保留原意、視覺升級。

---

## C. 寫程式 / 測試

### S09 Hermes Agent 回測 + 效能評估 `[本]`
- 觸發：Web UI 下「回測 Hermes」。
- 流程：PM 派發 → 本機跑回測 → 收集指標 → Code Review Agent 評估。
- 支柱：Manager + Hands。
- 產出：效能報告（含通過率、延遲、成本）。
- 驗收：完整跑完並產出可讀報告。

### S10 Playwright E2E 測試 + 精簡 log `[本]`
- 觸發：Web UI 或 CI 事件。
- 流程：本機喚醒 Playwright → 跑測試 → 壓縮 log → 回傳雲端。
- 支柱：Hands。
- 產出：精簡測試 log + 通過/失敗摘要。
- 驗收：log 精簡可讀、結果正確回傳。

### S11 Bug 修復 PR `[本]`
- 觸發：GitHub issue 或 Web UI。
- 流程：讀 issue → 改碼 → 跑測試 → 開 PR →（部署免審）。
- 支柱：Manager + Hands。
- 產出：PR + 測試報告。
- 驗收：PR 可合併、測試綠燈。

### S12 程式碼審查 + 風險評分
- 觸發：PR 事件。
- 流程：分析 diff → 風險評分 → 建議 reviewer → 產審查報告。
- 產出：Code Review 報告。
- 驗收：標出高風險變更與理由。

---

## D. 行政雜務

### S13 晨間 Digest `[排]`
- 觸發：每日排程。
- 流程：彙整 Email / 行事曆 / 新聞 / 待辦 → 摘要 → 語音 + ntfy 推播。
- 支柱：Manager + Guardrail（通知）。
- 產出：每日簡報（文字 + 語音）。
- 驗收：每早定時送達、內容正確。

### S14 Email 收件匣分類 / 摘要 / 提醒 `[排]`
- 觸發：排程輪詢。
- 流程：分類 → 標籤 → 摘要 → 需回覆者建提醒。
- 產出：分類後收件匣 + 待辦。
- 驗收：分類準確、重要信不漏。

### S15 行事曆排程協調
- 觸發：Web UI「幫我約會議」。
- 流程：查空檔 → 提時段 → 建立事件（含 CalDAV 同步）。
- 產出：行事曆事件。
- 驗收：無衝突、雙向同步成功。

### S16 週報表自動彙整 `[排]`
- 觸發：每週排程。
- 流程：彙整各來源數據 → 分析 → 產報表。
- 產出：data 表 + 文件報告。
- 驗收：數據正確、結論有依據。

---

## E. 研究 / 情報

### S17 競品分析
- 觸發：Web UI 給競品名單。
- 流程：多來源研究 → 比較 → 報告 → 存入 vault。
- 產出：競品分析報告 + vault 筆記。
- 驗收：來源可查、結構清晰、已歸檔。

### S18 市場 / 技術 Deep Research
- 觸發：Web UI 給主題。
- 流程：多跳研究 → 引用 → 綜整。
- 產出：附引用報告（doc）。
- 驗收：引用正確、論點扎實。

### S19 主題 / 競品動態監控 `[排]`
- 觸發：排程持續監控。
- 流程：定期掃描 → 偵測變化 → 有變化才 ntfy 推播。
- 支柱：Manager + Guardrail（通知）。
- 產出：變化警示。
- 驗收：有變化才通知、無雜訊。

### S20 研究蒸餾進 Vault Wiki `[排]`
- 觸發：研究完成後或排程。
- 流程：把研究成果轉原子筆記 → 建 backlink/tag → 寫入 `wiki/`。
- 支柱：Brain。
- 產出：vault wiki 筆記（持久知識）。
- 驗收：原子化、互連、不污染 `raw/` 與 `insights/`。

---

## F. 跨域基礎

- **Vault 維護**：自動 backlink / tagging / 孤兒連結偵測（僅整理，不亂寫 `insights/`）。
- **刪除攔截 `[審]`**：刪除 / 覆寫重要檔案或資料前，一律手機審核。

## 審核節點總表

| 動作 | 需審核 | 通道 |
|---|---|---|
| 寄出 Email / 客戶訊息 | 是 | ntfy Web Push |
| 花錢 / 大額預算 / 下單 | 是 | ntfy Web Push |
| 發布公開內容（貼文 / 網頁）| 是 | ntfy Web Push |
| 刪除 / 覆寫重要檔案 | 是 | ntfy Web Push |
| 部署網站 / push 程式 | 否 | — |
