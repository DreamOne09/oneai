## 已知 CLI / 平台限制(踩過的雷)
- **CLI 出包請改用 exe 直呼**:npm 安裝的 `zeabur` 是 ps1 wrapper,會吃掉 `--` 後參數、且讓 `template deploy` 靜默無效。直接呼叫 `…\node_modules\zeabur\zeabur_windows_amd64_v1\zeabur.exe` 一切正常(`template deploy`、`service exec -- …` 都 OK)。
- **資料庫(TCP)務必用 marketplace/模板,不要用自訂 Dockerfile**:自訂 Dockerfile 的 `EXPOSE` 被 zbpack 當成 HTTP「web 埠」,純 TCP(如 mongo 27017)會逾時不通。marketplace MongoDB(`template deploy -c KXL04P`)能正確宣告 TCP 27017 並附帳密/連線字串。
- **zbpack 會忽略「只有單一 Dockerfile」目錄的 Dockerfile,誤判成 caddy 靜態站** → 設服務變數 `ZBPACK_DOCKERFILE_PATH=Dockerfile` 強制使用(librechat 中招)。
- 跨服務 `${VAR}` 參照 CLI 有 bug → 內網串接用「服務內部主機名硬編」(如 `mongodb-noing.zeabur.internal:27017`)或 Dashboard。
- Zeabur 內網主機名 = `<服務名>.zeabur.internal`,改名不變;HTTP 服務注入 `PORT`(app 需綁 `$PORT`)。
- **多層引號/`$`(PowerShell→zeabur→sh→mongosh)**:把 JS 腳本 base64 後在容器內 `echo … | base64 -d > /tmp/x.js && mongosh --file` 執行,完全繞過引號地獄。

- **⚠️ 手動步驟(Zeabur Dashboard)**:
  1. **備份 Volume**:開啟 [oneai-backup 服務](https://zeabur.com/projects/6a36ad9046477d6038840b9d/services/6a36e0ac46477d603884113c) → 左側 `Storage` → `Add Volume` → 填寫 `/data/backups` → `Save`。掛載後備份才跨重啟持久保存。
